# 面向 VLM 智能体的装配分析思路沉淀

## 1. 文档目的

本文用于沉淀当前已经达成一致、且在工程上较为合理的思路，作为后续设计 VLM 装配分析智能体与软件数据接口的参考。

本文的讨论范围聚焦于：

- 输入为 `STEP/TSEP` 装配模型
- 当前软件的目标是为 VLM 提供可分析的数据，而不是单独完成最终装配规划
- 智能体希望分析的问题包括：
  - 谁和谁具有连接关系
  - 哪些零件可以先组成子装配
  - 哪个零件更适合作为基座
  - 哪些面或特征更适合夹持
  - 装配顺序、抓取策略、插入方向是否合理

## 2. 核心定位

当前软件最合理的产品定位不是“规则式装配规划器”，也不是“纯视觉装配识别器”，而是：

`VLM 智能体的数据供应层 + 假设验证层`

也就是说：

- 软件负责提供结构化几何、拓扑、候选关系、视觉证据和几何验证能力
- VLM 负责利用经验完成候选重排、关系解释、工艺偏好判断与多步推理
- 最终结果来自“VLM 假设 + 软件验证”的闭环，而不是单次视觉猜测

## 3. 当前判断：这条路线是否可行

结论：可行，而且优于纯规则路线。

原因如下：

1. 真实装配语义往往不是单个面与单个面的关系，而是“特征组与特征组”的关系。
2. 仅依赖 STEP/BREP 规则去识别基座、子装配、夹持面，需要大量强规则，维护成本高，泛化性差。
3. VLM 对“经验性判断”更有优势，例如：
   - 哪个零件更像底座/壳体/支架/紧固件
   - 哪几个零件更像应先形成子装配
   - 哪类区域更像可夹持区域
   - 哪种装配顺序更符合常见工艺经验
4. 但 VLM 不应直接替代几何事实，因为它无法天然保证：
   - 同轴性、配合间隙、干涉关系
   - 接近路径是否通畅
   - 夹具接近空间是否足够
   - 夹持是否避开螺纹面、密封面、配合面、薄壁面

因此，最合理的总体方案是：

`几何候选召回 -> 视觉证据采集 -> VLM 经验判断 -> 几何验证闭环`

## 4. 不应再坚持的旧抽象

当前最需要放弃的旧抽象是：

`把装配关系理解为 face pair`

这在简单模型中可用，但面对真实零件时会很快失效。典型问题包括：

- 一个孔可能由多个圆柱补片、端面、台阶面共同构成
- 一个螺栓的装配相关区域也往往不是一张面，而是一组面
- 某些夹持语义依赖一组平面、一个圆柱带、一个台阶组合，而非单张面

因此，后续所有分析都应优先围绕：

`feature group / function region / candidate relation`

而不是继续把单张 face 当作一等语义对象。

## 5. 新的推荐抽象层次

### 5.1 几何事实层

这是软件必须稳定提供的内容：

- assembly tree
- part 列表与层级
- part bbox / center / size
- face 几何摘要
- face normal / area / center
- geometry type
- cylinder axis / radius
- color map
- 当前可见状态、隔离状态、剖切状态

这一层不负责解释，只负责可信地描述模型。

### 5.2 候选层

这是规则或几何启发式最适合承担的职责：

- relation candidates
- feature-group candidates
- base candidates
- subassembly candidates
- grasp candidates
- insertion direction candidates

这一层不应输出唯一真值，而应输出一组带证据的候选。

### 5.3 证据层

这是 VLM 能力发挥的关键：

- beauty 多视图
- part-mask
- face-mask
- candidate overlay
- local crop
- section view
- isolated view

这一层要能把视觉图像和结构化对象一一对上。

### 5.4 验证层

这是闭环成立的关键：

- relation validity
- interference check
- insertion feasibility
- accessibility proxy
- grasp collision proxy
- stability proxy

这一步不替代 VLM，而是帮助智能体验证自己提出的假设。

## 6. VLM 在系统中的正确职责

VLM 最适合做的是：

- 经验性判断
- 模式识别
- 关系解释
- 候选重排
- 多步推理
- 结合视觉证据和结构化上下文做工艺偏好分析

VLM 不应单独承担：

- 纯几何真值判定
- 精确干涉分析
- 最终抓取可行性证明
- 最终装配可执行性证明

可以用一句话概括：

`VLM 负责“像不像”，验证器负责“行不行”。`

## 7. 关于基座、子装配、夹持面的判断

这些任务不适合完全写死规则，但也不能完全交给视觉猜。

更合理的方式是：

### 7.1 基座判断

软件输出基座候选及理由：

- 体积与支撑性
- 空间中心性
- 连接度
- 是否像紧固件或附件

VLM 基于结构化候选与视觉证据，判断：

- 哪个更像工艺基座
- 哪个更适合作为第一步稳定支撑件

### 7.2 子装配判断

软件输出：

- 零件间候选连接图
- 局部 feature-group 候选
- 内部连接强度与外部连接强度

VLM 判断：

- 哪几个零件更像应先预组装
- 哪些连接更像内部刚性关系

### 7.3 夹持判断

软件应输出的是“夹持候选特征组”，而不是只给原始 faces：

- 平行面组
- 圆柱夹持带
- 台阶支撑组
- 禁夹持面组
- 接近方向候选

VLM 再判断：

- 哪些区域更适合夹持
- 夹具类型更像什么
- 是否需要换姿态或二次装夹

## 8. 关于真实螺纹与复杂零件

真实螺纹会带来明显影响，必须提前考虑：

1. face 数量可能急剧增加
2. 颜色映射与 face-mask 会变得更碎
3. VLM 容易被高频细节吸引，忽略真正重要的装配语义
4. 螺纹面本身往往更像“禁夹持面”或“功能面”，而非推荐夹持面

因此，建议区分两种表示：

- `raw geometry`
  - 用于调试和局部精查
- `analysis geometry`
  - 用于 VLM 分析
  - 对螺纹等高频细节做区域化摘要或降权处理

一个实用原则是：

`全局分析优先看特征组与功能区域，局部验证再回到原始面级细节。`

## 9. 视觉证据应如何设计

### 9.1 不要只依赖一张图

VLM 证据必须是“证据包”，至少区分：

- 全局上下文图
- 候选局部图
- 遮挡消解图
- 夹持分析图

### 9.2 推荐的证据类型

#### 全局上下文图

- beauty 多视图
- part-mask 多视图

用途：

- 理解总装结构
- 理解零件分布
- 初步发现可能的子装配区域

#### 候选局部图

- isolated candidate parts
- candidate overlay
- local face-mask
- local color map
- 局部 crop

用途：

- 判断局部连接关系
- 判断 feature-group 是否合理

#### 遮挡消解图

优先级建议：

1. 隔离相关件
2. 剖切
3. 半透明上下文
4. 轻量爆炸图

原因：

- 隔离最稳定
- 剖切最适合内部孔腔和包裹结构
- 透明适合辅助上下文
- 爆炸图容易引入人为语义偏差，不应作为主证据

#### 夹持分析图

- 单件 isolated beauty
- 夹持候选 overlay
- 禁夹持面 overlay
- 接近方向箭头
- 装配环境中的 clearance 视图

用途：

- 判断夹持区域
- 判断接近空间
- 判断夹具类型偏好

### 9.3 对遮挡和包裹结构的建议

如果一个大部件完全包裹小部件，普通外部 beauty 图基本不够。

推荐顺序：

1. 隔离相关件
2. 对外壳剖切
3. 隐藏外壳或半透明显示外壳
4. 必要时生成轻量爆炸图

注意：

`mask 图尽量不要依赖透明叠加表达关系。`

更稳的方法是：

- 隐藏无关件
- 隔离候选件
- 剖切后再输出 mask

## 10. 智能体推荐工作流

建议后续 VLM 智能体遵循如下闭环：

1. 获取模型上下文
2. 获取候选关系和候选特征组
3. 选择高价值候选并请求证据包
4. 基于视觉证据和结构化上下文提出假设
5. 调用验证器检查假设是否成立
6. 若不成立，则继续请求新的视图或更换候选
7. 输出推荐方案、备选方案、证据和不确定性说明

推荐输出不是单一答案，而是：

- 主方案
- 备选方案
- 证据说明
- 风险点
- 需要补充的下一步证据

## 11. 当前软件最合理的短期目标

当前阶段，不建议让软件直接承担完整装配规划器角色。

更合理的短期目标是：

`把软件做成一个能为 VLM 智能体稳定提供结构化数据、视觉证据和验证能力的工作台。`

这意味着：

- 对外接口应更少、更高层
- 软件侧优先提供候选、证据、验证
- VLM 负责做经验分析和多步推理

## 12. 当前还需要补充的关键内容

后续建议重点补齐以下内容：

1. `feature-group` 抽象
2. `analysis geometry` 或细节降噪策略
3. 统一的 `evidence bundle` 规范
4. `validate_hypothesis` 验证能力
5. `grasp candidate` 数据层
6. 小规模评测集与标注样例

## 13. 一句话总结

当前最合理的方向不是“继续堆单面对单面规则”，而是：

`让软件提供可信的几何候选、视觉证据与验证能力，让 VLM 智能体基于这些数据完成装配关系、子装配、基座、夹持与顺序的自主分析。`
